EC2 provides a great deal of flexibility, allowing you to choose the instance type that fits your application needs. You can select instance types with various combinations of compute, memory, storage, and network power. Additionally, you can manage the instance lifecycle, including initializing, configuring, and stopping or destroying the instance when you are done using it.

You can also choose the geographic location where your EC2 instances run, so as to minimize latency and meet applicable data regulatory requirements. EC2 supports a variety of operating systems, including Linux, Windows, and macOS.

EC2 also includes additional features such as persistent storage with Amazon Elastic Block Store (EBS), secure networking with Virtual Private Cloud (VPC), and automatic scaling capabilities using Auto Scaling.

### Step 7 - Key Pair
A key pair in the context of AWS EC2 is a cryptographic key pair consisting of a public key and a private key. The public key is used to encrypt data, while the private key is used to decrypt data encrypted with the appropriate public key.

![Step 7](https://ik.imagekit.io/n1hqrcegw/articles/comprehensive-guide-creating-ec2-instances-aws-step-by-step/Frame_7.png?updatedAt=1684589515796)

In the EC2 context, key pairs are used to secure access to EC2 instances. When you launch your EC2 instance, you can generate or import key pairs. If you generate a key pair, AWS will generate a public and private key pair for you. The private key will be downloaded to your local computer, while the public key will be associated with the EC2 instance.

This private key is very important and should be properly guarded, as it provides access to your EC2 instance. If someone has access to the private key, they can log into your EC2 instance.

Key pairs are used in several scenarios in AWS EC2. Some common uses include:

1. Accessing the EC2 instance via SSH: In this case, the private key is used to authenticate when you connect to the EC2 instance via the Secure Shell (SSH) protocol.

2. Accessing EC2 instances via RDP (Remote Desktop Protocol): If you are using an EC2 instance with a Windows operating system, a key pair can be used to gain access to the instance via RDP.

3. Manage permissions to EC2 instances: Key pairs are also used to manage access permissions to EC2 instances. You can configure EC2 security groups to restrict access based on specific public keys.

In order to keep your EC2 instance secure, it is very important to keep the private key safe and not share it with insufficient parties.

## How to Connect With Instances Via SSH

Previously you have successfully connected directly to Ubuntu Server using the AWS Console with a browser and now we will try to connect Ubuntu Server using SSH via the terminal on our computer. Before starting make sure the terminal on your PC can use ssh, for Windows users I highly recommend you use [Powershell](https://learn.microsoft.com/en-us/powershell/), but if you use MAC OS / Linux you can follow the step by step below.

### Step 1
Make sure you are in the location of the folder where your key pair is stored.

![Step 20](https://ik.imagekit.io/n1hqrcegw/articles/comprehensive-guide-creating-ec2-instances-aws-step-by-step/Frame_20.png?updatedAt=1684589515796)

### Step 2
Execute the following command:

```bash
ssh -i example-key.pem ubuntu@18.136.120.248
```

**example-key.pem** is the name of your key pair file, **18.136.120.248** is the public ip address, you can check your IP address in the detailed Instance information.

![Step 21](https://ik.imagekit.io/n1hqrcegw/articles/comprehensive-guide-creating-ec2-instances-aws-step-by-step/Frame_21.png?updatedAt=1684589515796)

### Step 3 
Type the command yes if it appears like this.

![Step 22](https://ik.imagekit.io/n1hqrcegw/articles/comprehensive-guide-creating-ec2-instances-aws-step-by-step/Frame_22.png?updatedAt=1684589515796)

### Step 4
Don't panic if your terminal displays information like this, this information appears when you use SSH commands with a private key file (.pem or .ppk) that has too weak access permissions. This message indicates that your private key file access permissions are open to other users on the system.

To resolve this warning, you must fix the access permissions of your private key file so that it is read only by the owner (that is you) and not accessible by other users. On Linux or macOS systems, you can use the chmod command to set file access permissions, such as ``chmod 600 key.pem`` to grant read permissions only to the file owner.

"600" permission mode means that the file can only be read and written by the owner of the file. There are no access permissions for other groups or users on the system.

![Step 23](https://ik.imagekit.io/n1hqrcegw/articles/comprehensive-guide-creating-ec2-instances-aws-step-by-step/Frame_23.png?updatedAt=1684589515796)

So you can type the following command:

```bash
chmod 600 example-key.pem
```

![Step 24](https://ik.imagekit.io/n1hqrcegw/articles/comprehensive-guide-creating-ec2-instances-aws-step-by-step/Frame_24.png?updatedAt=1684589515796)

Then you can re-type the following command:

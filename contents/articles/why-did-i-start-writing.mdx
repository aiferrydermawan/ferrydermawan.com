Okay now you know that I am a Fullstack Web Developer, maybe you think I only handle Frontend and Backend, but the reality is not like that, I also handle DevOps, I handle Server, Testing, Security and fortunately AWS makes it easy for me to do all of that . How about the team? I don't have a team since I started working from 2020 until now.

Now you can imagine how much work I have to handle, I'm like superman, hahaha. Therefore it is important for me to take notes, be it through a note application, book or blog which I usually call second brain, because my brain is tired of having to remember all that. If I forget later I can look at my notes again. Okay, let's get back to the topic or title of this article "why did I start writing?" since there are some concerns that I'm having, I'll share a few points, first:

## As Documentation
With documentation people can understand how the code we write works quickly and easily, not only other people but also ourselves.

## To save time
Have you ever searched for a website with the same problem 2 times or more, even though previously you visited the website just because you didn't understand the code you copied, so you revisited the website, even though you had difficulty finding it again. So that it doesn't happen again, you can follow me to create a blog like this.

## Learning track record
Sometimes we don't know the limits of what we can learn and how long we have studied it, so we don't forget what we have learned we can record everything, so that later if we forget and want to remember again we can visit our notes again.

In conclusion Programmers should write because writing is one of the most effective ways to study, organize and store information. When a programmer writes, he must think deeply about what he wants to write and how he wants to convey that information.
